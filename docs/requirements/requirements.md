# togemaker（トゲメーカー）要件定義

## 1. コンセプト

アップロードした写真（ペット・人物・モノなど）の背景を自動で切り抜き、
アニメ『ガールズバンドクライ』で井芹仁菜が怒りや苛立ちを抱いたときの
**「トゲトゲ演出」（赤黒いトゲのオーラが背中・周囲から放射する表現）** の上に合成できる Web ツール。

参考プロダクト: **InspirationCat（キュピーン猫画像メーカー）**
動物写真をアップロード → ブラウザ内 AI で背景除去 → 集中線背景に合成 → サイズ調整・回転して保存、
という体験を「キュピーン」から「トゲトゲ」に置き換えたもの。

- 参考ツイート: https://x.com/nya3_neko2/status/2042107646157975651
- 技術解説記事: https://qiita.com/nogataka/items/a31fd9bbedb7adb4176c

## 2. 参考プロダクトから踏襲する設計思想

「サーバーに計算させない・保存させない・転送は CDN に任せる」

| 項目 | 内容 |
|---|---|
| AI 処理 | ONNX Runtime Web でブラウザ内実行（サーバー不要） |
| 実行バックエンド | WebGPU → WASM（マルチスレッド + SIMD）→ 素の WASM の自動フォールバック |
| モデル | ISNet 系（FP16 量子化で約 40MB）。初回 DL 後はブラウザキャッシュ |
| フロントエンド | Vite + バニラ TypeScript + Canvas API（フレームワークなし） |
| ホスティング | **Cloudflare Pages**（帯域幅無制限・無料。バズっても 0 円） |
| プライバシー | 画像はサーバーに送信されない（EXIF の GPS 情報等も外部に出ない） |

## 3. 機能要件

### MVP

1. **画像アップロード**: ファイル選択 + ドラッグ & ドロップ。モバイルはカメラロールから選択
2. **背景の自動切り抜き**: ブラウザ内 AI（ONNX Runtime Web）で被写体を抽出
3. **トゲトゲ背景への合成**: Canvas 上で「トゲトゲ背景レイヤー + 被写体レイヤー」を重ねて描画
4. **被写体の調整 UI**: ドラッグ移動・拡大縮小・回転（モバイルはピンチ / 2 本指回転に対応）
5. **PNG ダウンロード**: 合成結果を保存（X に貼りやすい 16:9 または 1:1 サイズ）
6. **進捗表示**: 初回のモデル DL（約 40MB）と推論中のローディング UI

### 拡張（v2 以降の候補）

- **怒りゲージ（トゲ強度スライダー）**: トゲの密度・大きさ・赤みを調整できる
  （原作でも感情の強さに応じてトゲの量が変化するため、「怒りの強さ → トゲの量」は原作準拠のマッピング）
- **ランダム再生成ボタン**: トゲの配置をシャッフルして好みの一枚を選べる
- **オーバーレイモード**: 切り抜かずに、写真の上（外周）からトゲトゲを重ねるモード
  （アニメの演出は「人物がシーン内にいて周囲にトゲが出る」ので、こちらの方が原作再現度は高い）
- **X 共有ボタン**: Web Share API / ツイート intent URL
- キャンバスサイズ選択（X ヘッダー用・アイコン用など）

## 4. 非機能要件

- **コスト 0 円**: Cloudflare Pages 無料プラン。サーバー・DB・ストレージを持たない
- **プライバシー**: 画像処理はすべてクライアント内で完結。アップロード画像を外部送信しない
- **会員登録・ログイン不要**: URL を開いて写真を選ぶだけ
- **モバイルファースト**: X からの流入はほぼスマホ想定。タッチ操作・縦画面レイアウト優先
- **初回ロード対策**: モデル DL はページ表示直後にバックグラウンドでプリフェッチ開始し、
  ユーザーが写真を選ぶ頃には準備完了している状態を目指す

## 5. 技術構成

| レイヤー | 技術 | 備考 |
|---|---|---|
| ビルド | Vite + TypeScript | フレームワークなし（参考プロダクト踏襲、バンドル最小化） |
| 背景除去 | `onnxruntime-web`(MIT) + ISNet(Apache-2.0) 自前実装 | `src/bg-removal.ts`。モデルは `npm run setup` で取得（下記 7. 参照） |
| 描画・合成 | Canvas API | トゲトゲ背景も Canvas で手続き生成（下記 6. 参照） |
| ホスティング | Cloudflare Pages | GitHub 連携で push → 自動デプロイ |
| モデル配信 | 同一オリジンから配信 | `Cache-Control` を長めに設定しブラウザキャッシュを効かせる |

## 6. トゲトゲ背景の実現方式

### 参考画像（アニメ演出）の視覚的特徴

- ベースはほぼ黒（わずかに暗い赤みのビネット）
- **赤・暗赤・黒の細長く鋭い破片（シャード）** が画面全体に飛び散る
- 中央（被写体まわり）は破片が少なく暗い無地、**外周ほど密度が高い**
- 破片にはモーションブラーがかかり、画面全体に軽い色収差（RGB ずれ）がある
- 原作設定: トゲは仁菜の背中・周囲から放射する怒りのオーラで、感情の強さに応じて量が変化する

### 案 A: Canvas で手続き生成（推奨）

- 黒ベース + 暗赤ビネットの背景に、細長い三角形／線分をランダムな角度・長さで多数描画
- 中心からの距離に応じて密度・サイズを上げる（中央は空ける）
- 破片ごとに軽いブラー、仕上げに全体へ色収差風の RGB シフト
- **メリット**: アニメのフレーム（著作物）を一切使わないため権利的に安全。
  ランダム生成で毎回違う絵になり、強度スライダー等のパラメータ化とも相性が良い

### 案 B: 自作の静止画素材

- 手描き or 生成した権利クリアな 1 枚絵を背景に敷く
- 手軽だが固定絵になり、強度調整・再生成ができない

→ **案 A を採用**。まず「トゲトゲ生成だけ」のプロトタイプを作って見た目を検証する。

## 7. 留意点・要確認事項

- **著作権**: 東映アニメーションの画像（スクリーンショット等）を素材としてそのまま使うのは NG。
  あくまで「風」の自前描画にとどめる。作品タイトル・ロゴの使用は避け、
  非公式・非営利のファンメイドである旨をフッター等に明記する
- **背景除去のライセンス**: `@imgly/background-removal` は AGPL-3.0 と確認（2026-07-13）したため不採用とし、
  `onnxruntime-web`(MIT) + ISNet(Apache-2.0, xuebinqin/DIS の重み・rembg 配布の ONNX 版) の自前実装を採用（案B）。
  品質は同等と検証済み。イマイチな点が出てきたら案A（AGPL で OSS 公開して @imgly 採用）を再検討
- **モデルサイズ**: int8 量子化で 179MB → 46MB（品質劣化なしを実写で検証済み）。
  Cloudflare Pages の 25MiB/ファイル制限には 16MiB×3 のチャンク分割配信で対応済み（2026-07-13）
- **モデル DL の離脱リスク**: プリフェッチ + 進捗バーで体感待ち時間を最小化する
- **iOS Safari**: WebGPU 非対応バージョンでは WASM フォールバックになるため、実機での速度検証が必要

