# 環境構築ガイド

## 1. 前提条件

| ソフトウェア | バージョン | 備考 |
|-------------|-----------|------|
| Node.js | 20 以上 | `scripts/setup-assets.mjs` が fetch / Web Streams を使用 |
| npm | Node 同梱 | |

データベース・環境変数・外部サービスの設定は**不要**（完全クライアントサイド）。

---

## 2. セットアップ

```bash
git clone <repository-url>
cd togemaker
npm install
npm run dev     # → http://localhost:5173/
```

AI モデル（int8 量子化・分割済み、`public/models/*.part*`）はリポジトリに含まれているため、
**追加のセットアップは不要**。`npm run setup` は fp32 元モデル（179MB）を `models-src/` に
取得するコマンドで、量子化パイプラインや品質比較のときだけ使う（deployment.md 参照）。

---

## 3. 開発コマンド

| コマンド | 内容 |
|---------|------|
| `npm run dev` | 開発サーバー（http://localhost:5173） |

---

## 4. 検証ページ・URL パラメータ

- **`/cutout.html`** — 背景除去の単体検証ページ（dev 専用・ビルドに含まれない）。
  `?auto` を付けると `public/test.local.png` で自動実行（任意のローカル画像を置く。.gitignore 済み）
- メインページの検証用パラメータ:
  - `?gauge=0-100` — 怒りゲージ初期値
  - `?seed=<16進>` — トゲトゲ背景のシード固定
  - `?aspect=16:9|1:1|9:16|3:1` — アスペクト比
  - `?demo` — `public/test.local.png` を自動で切り抜き合成（動作確認用）
  - `?threads=N` — WASM スレッド数の強制（トラブルシューティング用）
  - `?model=<URL>` — モデルの差し替え（例: `/models-src/isnet-general-use.onnx` で fp32 と品質比較）

## 6. トラブルシューティング

### Q: ポート 5173 で起動しない / ブラウザに反映されない
**A:** 別のプロセスが 5173 を掴んでいる可能性。`lsof -nP -iTCP:5173 -sTCP:LISTEN` で確認。
Vite はポートが塞がっていると自動で 5174+ に逃げるため、**ブラウザで開いている URL のポート番号**と起動ログの `Local:` が一致しているか必ず確認する。

### Q: 「WASM の初期化が 30 秒以内に完了しませんでした」
**A:** ブラウザ拡張機能や管理ポリシーの干渉が疑われる。シークレットウィンドウで再試行するか、`?threads=1` を付けてシングルスレッドで実行（実測 8s 程度）。

### Q: iPhone の写真（HEIC）が読み込めない
**A:** Chrome は HEIC を直接デコードできない。JPEG/PNG で書き出してから使用する。
