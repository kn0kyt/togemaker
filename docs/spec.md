## 1. プロジェクト概要

### 1.1 プロジェクト名

**togemaker（トゲメーカー）**

- リポジトリ名 / Cloudflare Pages プロジェクト名: `togemaker`（URL: `togemaker.pages.dev` 予定。プロジェクト作成時に空き確認）
- アプリ表示名: 未定（候補: トゲメーカー）
- ローカルフォルダは暫定 `toge`（後日リネーム可）
- 命名意図: 「トゲトゲ演出」は公式側でも使われる用語のため、`togetoge` 単体ではなく組み合わせ語にして非公式ツールとしての独立性を明確にした。ハッシュタグ `#togemaker` としても機能する

### 1.2 目的

アップロードした写真（ペット・人物・モノなど）の背景を自動で切り抜き、アニメ『ガールズバンドクライ』で井芹仁菜が怒りや苛立ちを抱いたときの「トゲトゲ演出」（赤黒いトゲのオーラが背中・周囲から放射する表現）風の背景に合成できる Web ツールを作る。

バズった個人開発プロダクト「InspirationCat（キュピーン猫画像メーカー）」のオマージュで、その設計思想「サーバーに計算させない・保存させない・転送は CDN に任せる」を踏襲し、**サーバー代 0 円** で運用する。

### 1.3 対象ユーザー

- 『ガールズバンドクライ』ファン（X などの SNS でネタ画像を作って共有したい人）
- 開発者自身（個人開発・学習目的）
- 想定流入はほぼ X からのモバイルユーザー

### 1.4 スコープ

**対応する範囲**
- ブラウザ内で完結する画像の切り抜き・合成・保存
- 静的サイトとしての公開（Cloudflare Pages）

**対応しない範囲**
- サーバーサイド処理・データベース・画像の保存や共有ストレージ
- アカウント機能・ログイン
- ネイティブアプリ

## 2. 機能一覧

### 2.1 主要機能

| # | 機能名 | 説明 | 優先度 |
|---|--------|------|--------|
| 1 | 画像アップロード | ファイル選択 + ドラッグ&ドロップ。モバイルはカメラロールから選択 | High |
| 2 | 背景自動切り抜き | ブラウザ内 AI（ONNX Runtime Web）で被写体を抽出 | High |
| 3 | トゲトゲ背景の生成・合成 | Canvas で手続き生成したトゲトゲ背景に被写体を重ねて描画 | High |
| 4 | 被写体の調整 UI | ドラッグ移動・拡大縮小・回転（モバイルはピンチ / 2本指回転） | High |
| 5 | PNG ダウンロード | 合成結果を保存（16:9 または 1:1） | High |
| 6 | モデル DL 進捗表示 | 初回の約 40MB モデルダウンロードと推論中のローディング UI | High |
| 7 | 怒りゲージ（トゲ強度スライダー） | トゲの密度・大きさ・赤みを調整。原作でも感情の強さに応じてトゲ量が変化する | Mid |
| 8 | ランダム再生成 | トゲの配置をシャッフルして好みの一枚を選べる | Mid |
| 9 | オーバーレイモード | 切り抜かず写真の外周からトゲトゲを重ねる（原作の演出に近い） | Mid |

| 11 | キャンバスサイズ選択 | X ヘッダー用・アイコン用などのプリセット | Low |

### 2.2 機能詳細

#### 機能1〜2: 画像アップロード & 背景自動切り抜き
- **概要**: 画像を選ぶと、ブラウザ内の AI モデルが被写体を切り抜く。画像は外部に送信されない。
- **ユーザーストーリー**: ファンとして、ペットや推しの写真を手軽にトゲトゲ画像にしたい。なぜなら SNS でネタとして共有したいから。
- **受け入れ条件**:
  - [ ] JPEG / PNG / WebP / HEIC（モバイル）を受け付ける
  - [ ] 切り抜き処理がブラウザ内で完結し、ネットワークに画像が送信されない
  - [ ] 処理中はローディング表示があり、UI がフリーズしない

#### 機能3: トゲトゲ背景の生成・合成
- **概要**: 黒ベース + 暗赤ビネットに、赤・暗赤・黒の細長く鋭い破片を Canvas でランダム描画。中央は空け、外周ほど密度を上げる。仕上げに色収差風の RGB シフト。
- **ユーザーストーリー**: ファンとして、「仁菜のトゲトゲ」だと一目で分かる背景にしたい。なぜなら原作の演出の再現度がツールの魅力そのものだから。
- **受け入れ条件**:
  - [ ] アニメのフレーム（著作物）を素材として一切使用していない
  - [ ] 生成のたびに配置が変わる（シード or 乱数）
  - [ ] 参考画像の特徴（黒ベース・赤黒シャード・外周ほど高密度・色収差）を再現している

#### 機能4〜5: 被写体の調整 & 保存
- **概要**: 切り抜いた被写体をキャンバス上で移動・拡縮・回転し、PNG として保存する。
- **ユーザーストーリー**: ユーザーとして、被写体の位置と大きさを直感的に調整したい。なぜなら構図が決まらないとネタ画像として成立しないから。
- **受け入れ条件**:
  - [ ] PC はマウスドラッグ + ホイール、モバイルはピンチ / 2本指回転で操作できる
  - [ ] 保存した PNG が X に投稿できる解像度・ファイルサイズである

## 3. 非機能要件

### 3.1 パフォーマンス
- 初回アクセス時に約 40MB の AI モデルを DL するため、ページ表示直後からバックグラウンドでプリフェッチし、進捗バーを表示する
- 2 回目以降はブラウザキャッシュから即起動（推論セッションはシングルトン化）
- 実行バックエンドは当面 WASM（マルチスレッド + SIMD）固定。WebGPU EP は ISNet の
  MaxPool(ceil_mode) 未対応（ort-web 1.27 時点）のため、対応後に有効化を再検討
- iOS Safari の実機で速度検証を行う

### 3.2 セキュリティ
- 画像処理はすべてクライアント内で完結し、画像・EXIF（GPS 等）を外部送信しない
- 認証・個人情報の取り扱いなし（サーバー・DB を持たない）
- 依存パッケージの脆弱性を定期確認（`npm audit`）。`.claude/rules/security.md` に準拠

### 3.3 可用性
- 静的サイトのため可用性は Cloudflare Pages（330 拠点以上のエッジ CDN）に依存。SLA 目標は設けない
- 帯域幅無制限の無料プランのため、バズによるアクセス急増でも費用は発生しない

## 7. 制約・前提条件

### 7.1 ビジネス制約
- 個人開発。運用費 0 円を維持する（Cloudflare Pages 無料プランの範囲内）
- 期限なし（趣味プロジェクト）
- **著作権**: 東映アニメーションの画像素材（スクリーンショット等）は使用しない。トゲトゲ表現は自前描画の「風」にとどめ、作品タイトル・ロゴの使用は避け、非公式・非営利のファンメイドである旨をフッター等に明記する

### 7.2 技術的制約
- サーバーサイドを持たない（静的ファイルのみ）。全処理がブラウザ内で完結すること
- AI モデルの初回 DL（int8 量子化済み 46MB）が必須（プリフェッチ + 進捗表示で離脱を抑える）
- **Cloudflare Pages の 1 ファイル 25MiB 制限**には、モデルの 16MiB×3 チャンク分割配信で対応済み
  （ブラウザで並列 fetch → 結合。再生成手順は deployment.md）
- マルチスレッド WASM に COOP/COEP ヘッダーが必要（dev は vite.config.ts、本番は `public/_headers`）
- iOS Safari の WebGPU 非対応環境では WASM フォールバックとなり速度が落ちる
