# コードレビュールール（共通）

コードレビュー時に確認すべき観点をまとめています。

---

## レビュー観点

### 1. 機能要件
- [ ] 要件を満たしているか
- [ ] エッジケースは考慮されているか
- [ ] エラーハンドリングは適切か

### 2. コード品質
- [ ] 命名は明確で一貫性があるか
- [ ] 関数・クラスの責務は単一か
- [ ] 重複コードはないか
- [ ] 適切なコメントがあるか（過剰でも不足でもない）

### 3. パフォーマンス
- [ ] N+1問題はないか
- [ ] 不要なデータ取得はないか
- [ ] 適切なキャッシュ戦略か

### 4. テスト
- [ ] テストは書かれているか
- [ ] 正常系・異常系がカバーされているか
- [ ] テストは独立して実行可能か

### 5. セキュリティ
- [ ] security.md のチェックリストを確認

---

## レビューコメントの書き方

### 良いコメント例

```
[must] N+1クエリが発生しています。Eager Loadingを使用してください。
該当箇所: UserController.php:45

[suggest] この処理はServiceクラスに切り出すと再利用しやすくなります。

[question] この条件分岐の意図を教えてください。
```

### プレフィックス

| プレフィックス | 意味 |
|--------------|------|
| `[must]` | 必ず修正が必要 |
| `[should]` | 強く推奨 |
| `[suggest]` | 提案（任意） |
| `[question]` | 質問・確認 |
| `[nit]` | 細かい指摘 |
