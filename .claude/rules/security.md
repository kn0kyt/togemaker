# セキュリティルール（共通）

実装・レビュー時に必ず確認する。ブランチ単位のまとまった変更には組み込みの `/security-review` も活用する。

## チェックリスト

### 入力・出力
- ユーザー入力はサーバー側で必ずバリデーション（クライアント側検証だけに依存しない）
- SQL はプレースホルダ / ORM 経由（文字列連結は禁止）
- HTML 出力はエスケープ、API レスポンスは適切な Content-Type
- ファイルアップロードは拡張子・MIMEタイプ・サイズを検証

### 認証・認可
- 全エンドポイントで認証・認可をチェック（IDOR 対策として所有者確認を忘れない）
- パスワードは bcrypt / Argon2 でハッシュ化、セッション ID は認証後に再生成
- ログイン等にはレート制限（ブルートフォース対策）

### データ保護
- 機密情報のハードコーディング禁止（.env / シークレットマネージャを使用）
- ログ・エラーメッセージに機密情報や内部実装の詳細を出力しない
- 通信は HTTPS、Cookie は HttpOnly / Secure 属性を設定

### 依存関係
- 依存パッケージの脆弱性を定期確認（`npm audit` / `composer audit` / OWASP Dependency-Check 等）
